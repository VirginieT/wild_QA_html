Présentation du métier d'automaticien



Rôle du testeur automaticien

L'automaticien de test a plusieurs casquettes : celle detesteur mais aussi celle de développeur. Entre toutes les plateformes d'automatisations et les multiples langages de programmation, l'automaticien doit être curieux, et sa soif d'apprendre insatiable. Il automatise tout ce qui peut l'être, tests fonctionnels, tests d'API, tests de performance, tests data... à condition d'y trouver de la valeur métier.

En tant que Testeur automaticien, vous réaliserez la stratégie des tests automatisés dans le plus grand respect des exigences client, en intégrant l'utilisateur final au centre de sa réflexion. Vous interviendrez de manière autonome sur des projets IT variés et parfois complexes, en assurant une communication et un suivi régulier auprès des équipes projets (métiers, développeurs).



Compétences les plus recherchées

  1. Connaissance des technologies webservice et mobile (smartphones / tablettes / objets connectés)
  2. Maîtrise d'un outil d'automatisation de test Web (Front ou Backend)
  3. Maîtrise d'un outil de gestion des tests et anomalies
  4. Maîtrise des méthodologies de test (certification ISTQB fondation alors là vous êtes parfait !)
  5. Expérience dans la mise en oeuvre de plans de tests (étude des spécifications, rédaction, exécution, suivi)


Exemples de compétences appréciées

  • Connaissances en développement et scripting : JAVA, Junit, Cucumber, JavaScript, Bash, XML, JSON
  • Outils de supervision : Kibana, Prometheus, Logstash, MySyslog
  • Outils de tracing de logs : Kibana, Firebase etc.
  • Connaissances réseau : Ethernet, IP, TCP, UDP, SSH, VoIP (SIP)
  • Utilisation d'un serveur TFS (ADS) pour lancer les test-automatisés
  • Connaissance de différents langages : JAVA, JavaScript, html, jquery
  • Compréhension des fichiers :.bat,.sh,.xml,.json,.csv
  • Autres outils d'automatisation et de tests Api Rest (Soap UI, Jmeter, Xamarin Test Cloud)

Les outils d'automatisations du testeur automaticien

Selenium

Selenium est un outil portable et open source qui fournit des tests fonctionnels de divers modules d'application Web sur les plates-formes et les navigateurs. Il prend en charge l'exécution de texte parallèle, ce qui économise beaucoup de temps de test. Selenium prend en charge de nombreux langages comme Java, Ruby, C #, Python, etc.

Selenium exécute un outil de lecture pour enregistrer les tests fonctionnels, vous n'avez donc pas besoin d'apprendre le langage de script de test. Le sélénium est très facile à installer; vous devez ajouter le plugin spécifique au navigateur. Il automatise les navigateurs Web en donnant une vitesse et un niveau d'efficacité différents qui ne sont pas possibles grâce à des tests manuels.

EggPlant

Eggplant est l’un des meilleurs outils de d’automatisation des tests pour les applications de bureau et les tests d’interface graphique.

Testplant a développé eggPlant pour que les testeurs puissent effectuer différents types de tests.

Cucumber

Cucumber est une infrastructure de développement de type BDD (Behavior Driver Development) qui permet de développer nos applicatifs et de concevoir des tests fonctionnels automatisés en les rendant compréhensible et lisible par tous les acteurs de notre projet.

Cucumber est un outil permettant de développer à partir des spécifications du logiciel. Les spécifications sont ainsi utilisées comme base de développement pour renforcer la collaboration entre les équipes, étant un document lisible et compréhensible par tous.

Cet outil est particulièrement efficace pour concevoir des tests d’acceptation sur les applications web.



Aptitudes relationnelles et comportementales visées

  • Doté d'un excellent relationnel (oral et écrit), vous êtes autonome sur vos projets
  • Votre flexibilité et votre esprit d'équipe vous permettront une bonne intégration au sein du projet
  • Vous êtes force de proposition dans la réalisation de tes tâches
  • Organisé et rigoureux, vous appliquez les bonnes pratiques du test.

Conclusion

Etre un bon automaticien testeur, c'est trouver les bugs avant qu'ils ne deviennent des papillons.

C'est comme être un super-héros, mais pour les logiciels.
